# Проект виджета банковских операций 
## Цель проекта: написать виджет, позволяющий обеспечить удобное и быстрое взаимодействие клиента с системой банка


## Установка:

1. Клонируйте репозиторий:
```
git clone https://github.com/username/project-x.git
```
2. Установите зависимости:
```
pip install -r requirements.txt
```
3. Вызовите файл main.py:
```
python main.py
```

Функционал программы написан в пакете src, каждый модуль отвечает за определенную часть программы, в каждой функции присутствует docstring

## Тестирование
В программе реализованы тесты, под каждый функционал.
Для каждого модуля написан отдельный тестовый модуль.
Чтобы провести тестирование, выполните команду:
```
pytest
```
Или можно воспользоваться средствами IDE Pycharm.

## Дополнительный функционал
### Импорт функций из модуля:
```
python
from src.module_name import filter_by_currency, transaction_descriptions
```
### Функция filter_by_currency 
Назначение: Отфильтровывает транзакции по указанному типу валюты.
Аргументы: 

    - transactions: список словарей с транзакциями.  
    - currency_code: строка, обозначающая код валюты (например, "USD").
Возвращает: итератор, выдающий транзакции, совпадающие с указанным типом валюты.

### Функция transaction_descriptions 
Назначение: Генерирует описания транзакций
Аргументы: 

    - transactions: список словарей с транзакциями.
Возвращает: итератор, выдающий описания транзакций.

### Функция card_number_generator
Назначение: генерирует номера банковских карт в диапазоне от start до end включительно.

Ограничен значением 9999 9999 9999 9999.

Аргументы:

    - start: начальное значение диапазона генерации
    - end: конечное значение диапазона генерации
Возвращает: генератор, который выдает номера банковских карт в формате XXXX XXXX XXXX XXXX , где X — цифра номера карты

## Логирование
### Декораторы обеспечивающие логирование 
В проекте реализован дополнительный функционал, который обеспечивает логирование результатов выполнения функций.
При этом в зависимости от переданного параметра можно выбрать куда будут записываться логи: в файл или в консоль.
Найти код декораторов можно в модуле decorators.py 

### Тестирование декораторов
Также в проекте добавлен модуль test_decorators.py
В нем прописаны тесты для декоратора, проверяющие корректность его работы, а также результата декорируемой функции.

## Чтение данных из файлов
Реализован функционал чтения данных из csv- и excel-файлов
Для реализации данного функционала были написаны две функции в модуле file_reader.py
Для проверки корректности работы данных функций были написаны тесты в модуле test_file_reader.py
Для запуска тестов из командной строки воспользуйтесь командой
```
pytest ./src/test_file_reader.py
```
